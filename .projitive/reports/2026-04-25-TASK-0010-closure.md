## 变更摘要

实现多目的地备份支持：单次数据库导出后，自动将同一备份产物并行上传到多个目的地。

### 核心变更

| 文件 | 变更 |
|------|------|
| `source/core/types.ts` | `BackupDestination` 新增 `name` 字段；`BackupConfig` 支持 `destinations[]` 与 `destination` 互斥 |
| `source/core/scanner.ts` | 校验逻辑：单/多目的地互斥检测 |
| `source/cli/run.ts` | 解析 `destinations[]` 配置；`createMultiStorageDriver` 实现多路 fan-out |
| `source/core/executor.ts` | `generateFileKey` 支持多目的地；`stageUpload` 收集每个目的地上传结果到 `BackupResult.uploadResults` |

### 新增类型

```typescript
// BackupResult 新增
uploadResults?: UploadResult[]  // 每个目的地的上传结果

// BackupDestination 新增
name?: string  // 目的地命名（可选）
```

### 架构设计

```
单次Dump → 同一 StagedFile → 并行上传到 N 个目的地
                                ↓
                         MultiStorageDriver
                      (Promise.all fan-out)
                                ↓
                    [dest-0] [dest-1] ... [dest-N]
                    UploadResult[0..N] → BackupResult.uploadResults
```

### 日志输出（多目的地）

```
Uploading backup { destination: "2 destinations", key: "...", size: 14 }
[primary] Uploading backup { key: "...", size: 14 }
[primary] Upload completed { key: "...", size: 14 }
[secondary] Uploading backup { key: "...", size: 14 }
[secondary] Upload completed { key: "...", size: 14 }
All destination uploads completed { destinations: [{ name: "primary", type: "s3" }, ...], key: "...", size: 14 }
```

### 配置示例

```yaml
name: prod-backup
source:
  type: postgresql
  database: app

destinations:
  - name: primary
    type: s3
    s3:
      endpoint: https://s3.amazonaws.com
      region: us-east-1
      bucket: prod-primary
  - name: secondary
    type: s3
    s3:
      endpoint: https://s3.cn-north-1.amazonaws.com.cn
      region: cn-north-1
      bucket: prod-secondary
```
