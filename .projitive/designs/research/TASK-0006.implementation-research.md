# TASK-0006 Implementation Research Brief

## Task
实现 Prometheus Metrics 上报

## Summary
实现 Prometheus 指标上报，支持备份成功/失败/耗时/大小等指标

---

## Design Guidelines and Specs

### Prometheus Text Exposition Format
- 标准 Prometheus `/metrics` 端点格式
- 使用 Prometheus 客户端库或手动实现

### Integration Points
- BackupExecutor - 记录备份成功/失败计数
- BackupResult - 提取 duration, size 等指标

---

## Code Architecture and Implementation Findings

### 1. Metrics Module
- 新建 `source/metrics/` 模块
- `source/metrics/index.ts` - 导出
- `source/metrics/collector.ts` - 指标收集器
- `source/metrics/types.ts` - 指标类型定义

### 2. Metrics Types
```typescript
// Counter: 备份成功/失败次数
// Histogram: 备份耗时分布
// Gauge: 最后一次备份大小
```

### 3. HTTP Endpoint
- 添加 `/metrics` 端点到健康检查服务器
- 或单独创建 metrics 服务器

### 4. BackupExecutor Integration
- 在 executeTo 完成后调用 metrics collector

## Prometheus Metrics Spec

| Metric Name | Type | Labels | Description |
|------------|------|--------|-------------|
| `backup_total` | Counter | `status=success\|failed` | 备份总次数 |
| `backup_duration_seconds` | Histogram | `task_name` | 备份耗时 |
| `backup_size_bytes` | Gauge | `task_name` | 最后一次备份大小 |
| `backup_last_success_timestamp` | Gauge | `task_name` | 最后成功时间 |
