## Implementation Plan

### Phase 1: Extend StorageDriver Interface
**File**: `source/core/interfaces.ts`
- 新增 `StorageObject` 类型：
  ```typescript
  interface StorageObject {
    key: string
    size: number
    lastModified: Date
  }
  ```
- 新增 `StorageDriver.list(prefix?: string): Promise<StorageObject[]>`

### Phase 2: Implement S3 listObjects
**File**: `source/adapters/storage/s3.ts`
- 新增 `listObjects(prefix?: string): Promise<StorageObject[]>`
- 使用 `ListObjectsV2Command`，支持分页
- 过滤掉文件夹（CommonPrefixes）

### Phase 3: Create RetentionExecutor
**File**: `source/retention/index.ts`, `source/retention/executor.ts`
- `RetentionExecutor` 类
- `applyRetention(config: ResolvedConfig): Promise<RetentionResult>`
- 流程：
  1. 从 config 读取 `retentionDays`
  2. 调用 storageDriver.list() 获取所有文件
  3. 解析 key 提取日期（通过 key 中的日期部分）
  4. 过滤超过 retentionDays 的文件
  5. 逐个调用 storageDriver.delete() 删除

### Phase 4: Integrate with CLI
**File**: `source/cli/run.ts`
- 新增 `backup retention --config <path>` 命令
- 支持 `--dry-run` 参数（只显示待删除文件）

### Phase 5: Add Tests
**File**: `source/retention/executor.test.ts`
- 测试正常删除过期备份
- 测试保留策略为 0 时不删除
- 测试 dry-run 模式

## Risks and Open Questions

1. **Q: 文件名中没有日期怎么办？** → 依赖 key 中的日期提取（{{.Date}}），如果没有日期用 LastModified
2. **Q: 保留策略为 0 怎么处理？** → 视为不自动清理
3. **Q: list 时分页处理？** → 使用 ListObjectsV2 的 IsTruncated 循环
4. **Q: 保留策略基于 S3 LastModified 还是文件名中的日期？** → 优先用文件名日期，fallback 到 LastModified
