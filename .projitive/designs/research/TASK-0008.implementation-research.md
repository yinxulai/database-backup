实现健康检查接口

## Summary

## Design Guidelines and Specs

### CLI Architecture
- `source/cli/run.ts` - CLI 入口，支持 `run` / `validate` 命令
- 需要新增 `serve` 命令启动 HTTP 服务

### Node.js Built-in
- `http` module - 内置 HTTP 服务器，无需额外依赖

---

## Code Architecture and Implementation Findings

### 1. Health Check Server
- 使用 Node.js `http` 内置模块
- 路径: `source/health/` (新模块)
  - `index.ts` - 导出
  - `server.ts` - HTTP 服务器实现
  - `handlers.ts` - 处理器
  - `types.ts` - 类型定义

### 2. Health Endpoints
| Path | Probe | Description |
|------|-------|-------------|
| `/health/live` | Liveness | 进程存活，always 200 |
| `/health/ready` | Readiness | 服务就绪（DB 连接正常）|
| `/health` | Summary | 汇总状态 |

### 3. Readiness Check
- `source/core/interfaces.ts` - `DatabaseDriver.testConnection()` 已存在
- 复用现有连接测试逻辑

### 4. CLI Integration
- `source/cli/run.ts` - 新增 `serve` 命令
- `backup serve --port 8080 --config backup.yaml`
